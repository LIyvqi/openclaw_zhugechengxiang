## 遇到的问题及解决方案

### 问题 1：GitHub Token 认证失败
**现象：**
- `remote: Invalid username or token`
- `Authentication failed`

**原因：**
- Token 格式/权限问题

**解决：**
- 重新生成 Token，确保勾选 `repo` 权限

---

### 问题 2：GitHub Push Protection 阻止推送
**现象：**
```
remote: error: GH013: Repository rule violations found
Push cannot contain secrets
```

**原因：**
- 备份中包含敏感信息（Token、API Key）

**解决：**
- 方案 A：在 GitHub 后台允许泄露的 Token
  - 链接：`https://github.com/{user}/{repo}/security/secret-scanning/unblock-secret/{id}`
- 方案 B：重新创建仓库（更干净）

---

### 问题 3：敏感信息暴露
**现象：**
- .env 文件包含 Token 明文

**解决：**
- 备份脚本过滤敏感信息：
```bash
grep -v "API_KEY\|TOKEN\|SECRET\|PASSWORD" /root/.openclaw/.env > backup/.env
```

---

### 问题 4：嵌入的 Git 仓库
**现象：**
```
warning: adding embedded git repository
```

**原因：**
- skills 目录包含 .git 子目录

**解决：**
- 备份时删除 .git 目录：
```bash
find "$REPO_DIR/skills" -name ".git" -type d -exec rm -rf {} +
```

---

## 经验总结

1. **Token 权限** - 需要 `repo` 完整仓库权限
2. **Push Protection** - GitHub 会自动扫描敏感信息，需要在后台允许或重新创建仓库
3. **敏感信息** - 备份前过滤，不要上传明文 Token
4. **Git 子目录** - 复制前删除 .git 目录避免嵌入仓库问题
